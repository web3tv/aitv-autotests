# Aspect-ratio test videos

10 вариантов одного ролика с разными соотношениями сторон — для тестов плеера/шортсов
на нестандартных форматах. Источник: `../shortsVideo.mp4` (720×1280, 9:16, без аудио),
конвертация ffmpeg: `scale=W:H:force_original_aspect_ratio=increase,crop=W:H`
(масштаб с заполнением + центр-кроп), libx264, crf 23. ~9 МБ суммарно.

| Файл | Ratio | Разрешение |
|---|---|---|
| AR_9x16_720x1280.mp4 | 9:16 | 720×1280 |
| AR_9x20_720x1600.mp4 | 9:20 | 720×1600 |
| AR_2x3_720x1080.mp4 | 2:3 | 720×1080 |
| AR_3x4_720x960.mp4 | 3:4 | 720×960 |
| AR_4x5_720x900.mp4 | 4:5 | 720×900 |
| AR_1x1_720x720.mp4 | 1:1 | 720×720 |
| AR_16x9_1280x720.mp4 | 16:9 | 1280×720 |
| AR_4x3_960x720.mp4 | 4:3 | 960×720 |
| AR_2x1_1440x720.mp4 | 2:1 | 1440×720 |
| AR_21x9_1680x720.mp4 | 21:9 | 1680×720 |

NB: при загрузке через API бэкенд сам присваивает тип по соотношению сторон:
≤ 1:1 (вертикальные + квадрат) → `short`, шире 1:1 → `video`. Явно передать тип
через API нельзя (форма `/videos/{id}` его не принимает). Для вертикальных файлов
в `VideoApi.uploadVideo` передавайте `contentType: 'short'`, иначе `waitForProcessing`
таймаутится, поллируя не тот тип.
